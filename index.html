Slack 병행 테스트의 목적

단순 메신저가 아닌, AI·시스템 통합 인터페이스로서의 가능성 검증

🤖 AI 에이전트 통합 허브

  • Slack은 글로벌 기업에서 AI 비서의 대화 창구로 활용 중
  • Salesforce(모회사)의 Agentforce, OpenAI ChatGPT 등 AI 에이전트가 Slack 안에서 직접 동작
  • 직원이 채널에서 질문하면 AI가 사내 데이터를 검색·요약·답변
  • 광동 활용: 논문 요약, 시장 데이터 조회, 보고서 초안 생성을 Slack 채널 안에서 수행

🔗 시스템 연동 플랫폼

  • ERP·MES·SFA 등 기존 시스템의 알림과 워크플로우를 한 곳으로 통합
  • 공장 장애 → 자동 알림 → 스레드 대응 → 화상회의(Huddle) 즉시 전환
  • 외부 파트너와 Slack Connect로 보안 유지하며 협업
  • 광동 활용: 현장 이슈 실시간 대응, 대행사 협업 채널 운영

검증 계획 (PoC)

"전사 도입 전에, 제한된 범위에서 먼저 증명합니다"

PoC 개요

기간준비: 3.23~31 / PoC: 4.1 ~ 5.27 8주
대상 부서CMO본부 (95명) + 기획관리본부 (60명)
총 참여 인원155명
라이선스GWS Enterprise Standard + Slack Business+
PoC 비용트라이얼 협의 중 (별도 산출)

CMO본부 선정 근거

  • 영업·마케팅·기획·디자인 등 다양한 직무 영역을 한 본부 안에서 커버
  • 직무별 협업 패턴이 달라 도구 적합성 검증 범위가 넓음
  • 현장(영업)↔본사(기획) 간 실시간 소통 니즈 최대

기획관리본부 선정 근거

  • 전사 KPI 취합·보고 업무 집중
  • 부서간 협업 빈도 최고
  • 전사 거버넌스 통합 관리 검증 적합

부서별 핵심 검증 시나리오

각 부서의 실제 업무를 새로운 도구로 수행하고, AS-IS 대비 변화를 측정합니다

부서검증 시나리오현재 방식 (AS-IS)PoC 방식 (TO-BE)
전략기획 경영보고서 공동 작성 각 부서 PPT 메일 회신 → 수작업 합본 Slides 공동편집, 합본 작업 제거
영업지원 견적서·계약서 검토 메일 첨부 핑퐁, 최종본 혼선 Docs 제안 모드, 버전 자동 관리
총무 사내 규정·매뉴얼 관리 규정 변경 시 메일 배포, 구버전 혼재 Docs 단일본 관리, 변경 이력 자동 추적
디자인 시안 파일 버전 관리·피드백 메일 첨부 시안 반복, 잘못된 버전으로 작업 Drive 단일 버전 + 스레드 피드백, 버전 오류 원천 차단
F&B영업 거래처 미팅 기록 + 프로모션 조율 개인 노트, 온/오프 정보 분리 모바일 Docs 즉시 작성, 통합 캘린더
브랜드전략 캠페인 기획~실행~분석 부서 순차 메일 전달, 정보 단절 프로젝트 채널에서 전 부서 동시 진행
CMO전략 주간 보고 취합 + 예산 관리 개별 PPT 메일 → 수작업 취합, 월말 예산 사후 인지 Slides 동시 편집, Sheets 실시간 예산 대시보드

크로스 본부 검증 시나리오

부서 내 개선뿐 아니라, 본부 간 협업이 얼마나 빨라지는지를 핵심적으로 검증합니다

📊 A. 월간 경영보고 프로세스

전략기획 + CMO전략 + 각 본부

🎨 B. 디자인 에셋 협업 프로세스

브랜드전략 + 디자인 + 영업 + CMO전략

요청 양식 통일 → Drive 단일 버전 관리 → 스레드 피드백으로 메일 버전 오류 제거

검증: 파일 버전 오류 건수, 시안 피드백 리드타임

📝 C. 거래처 계약·견적 프로세스

🚀 D. 캠페인 기획~실행 통합

브랜드전략 + 디자인 + 영업 + CMO전략

프로젝트 채널에서 기획→크리에이티브→영업→실행→분석까지 전 부서 동시 진행

검증: 기획~론칭 리드타임, 부서 간 정보 누락 건수
